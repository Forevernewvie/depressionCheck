# Map Readability Hardening Prompt

다음 기준으로 `MapScreen` 의 가독성과 유지보수성을 더 높여라.

## 목표
- `MapScreen` 을 화면 조립 전용에 가깝게 단순화한다.
- 로드 카드, 필터/정렬 카드, 빈 상태 카드, 마커 생성 로직을 화면 본체에서 분리한다.
- 동작 변경 없이 읽기 쉬운 구조를 우선한다.

## 우선 원칙
- 가독성 > 미세 최적화
- 하드코딩 금지
- 중복 로직 금지
- 신규/수정 함수에 목적 설명 주석 추가
- 이미 존재하는 정책(`MapPresentationPolicy`, `MapViewController`)과 역할이 겹치지 않게 한다

## 실행 절차
1. `MapScreen` 의 반복 카드 UI 를 전용 위젯으로 추출한다.
2. 마커 생성 로직을 순수 팩토리 또는 헬퍼 클래스로 분리한다.
3. 화면 본체는 상태 조회, 섹션 조립, 액션 연결만 담당하게 만든다.
4. 관련 테스트를 추가하거나 보강한다.
5. `flutter analyze` 와 관련 테스트 또는 전체 테스트를 실행한다.

## 검증 포인트
- 로드 카드의 상태별 CTA 와 안내문이 유지되는가
- 필터/정렬 조작이 기존처럼 동작하는가
- 빈 상태 문구가 정책과 동일하게 유지되는가
- 마커 수와 위치가 기존과 동일한가
- `MapScreen` 파일이 이전보다 즉시 읽기 쉬워졌는가
