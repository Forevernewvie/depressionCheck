# SEO / i18n 제한사항 메모

## 현재 상태
- 앱은 ko/en을 런타임 설정으로 지원하지만 URL 경로가 언어별로 분리되어 있지 않습니다.
- 따라서 `hreflang`은 동일 URL(홈) 기준으로 선언됩니다.

## ASSUMPTION
- 현재 MVP에서는 콘텐츠 URL 분리보다 앱 기능 안정성을 우선합니다.

## 다음 단계 권장
1. 언어별 정적 랜딩 경로 도입 (`/ko`, `/en`)
2. 각 경로에 맞는 `hreflang`/canonical 분리
3. landing 페이지에서 앱 딥링크(`/?lang=ko`) 연결
4. 검색봇 친화적인 정적 콘텐츠(앱 소개/안전 고지/FAQ) 보강
